How this site is built

This portfolio is its own case study: a small Angular 17 application that uses the modern toolkit the way I use it at work — just with the hood left open.

Signals everywhere

Every piece of local state is a signal: the theme, the active section, the project filter, even the experience counter in the hero. Derived state like the filtered project list is a computed(). Press i anywhere to open the signal monitor and watch the values update as you interact with the page.

Standalone & OnPush

There is not a single NgModule in the codebase. Every component is standalone, declares exactly what it imports, and runs with ChangeDetectionStrategy.OnPush. Signals make that combination feel effortless — change detection runs only where state actually changed.

Lazy by default

Routes load via loadComponent, and every section below the hero sits inside a @defer block. Sections hydrate on idle so deep links and the navigation anchors keep working, while the initial bundle stays focused on the first paint.

CSS does the heavy lifting

Reveal animations, the tilt on project cards, the inspect-mode overlay, the theme change — all CSS, driven by classes and custom properties. JavaScript only flips signals. Motion respects prefers-reduced-motion across the board.

Typed content, no CMS

Every word on this site lives in typed constants under src/app/data, validated by interfaces in src/app/models. Updating the portfolio is a pull request, the way it should be.
